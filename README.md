# VRCTI

1. 지표 구조 (4진법 조합)

각 질문의 점수를 합산하여 4가지 척도 중 하나를 결정함:

- E (Explorer) vs I (Inside): 퍼블릭/새로운 만남 vs 지인/프라이빗 공간 선호
- D (Deep) vs G (Gamer): 감정적/신체적 깊은 몰입 vs 시스템적/관조적 태도
- F (Friendly) vs C (Creator): 정서적 교류/친목 vs 아바타 제작/사진 촬영 등 창작 활동
- S (Sync) vs P (Persona): 현실 자아 투영 vs 캐릭터 페르소나 유지

2. 핵심 기능 요구사항

설문 방식: 각 지표당 O/X 질문.

결과 계산:

점수가 중앙값보다 높으면 A코드, 낮으면 B코드.

디자인:

VRChat 느낌이 나는 다크모드 기반의 네온/사이버네틱 디자인.

결과 화면에는 해당 유형의 별명(예: "심해형 개발자", "퍼블릭 인싸")과 간단한 설명 표시.

공유 기능: 결과 페이지를 이미지로 캡처하거나 결과 텍스트를 클립보드에 복사하는 버튼 포함.

(D/G) "나는 가상 세계에서 누군가 나를 만지면 실제로 감각이 느껴지는 것 같다."

(E/I) "나는 모르는 사람이 많은 퍼블릭 월드에 가는 것이 즐겁다."

(F/C) "나는 VRChat에 접속하면 사람들과 대화하기보다 유니티나 SDK 작업을 더 많이 한다."

(S/P) "나는 인게임에서도 내 실제 목소리와 플레이 성향을 그대로 사용하는 편이다."

4. 출력물
